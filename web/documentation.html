Documentation

Guide complet d'utilisation du système IDS ML

Installation

1. Cloner le Dépôt

git clone https://github.com/votre-repo/ids-ml.git
cd ids-ml

2. Installation avec Script

Linux/Mac:

chmod +x setup.sh
./setup.sh

Windows:

setup.bat

3. Installation Manuelle

# Créer un environnement virtuel
python -m venv venv
source venv/bin/activate  # Linux/Mac
# ou
venv\Scripts\activate  # Windows

# Installer les dépendances
pip install -r requirements.txt

Utilisation

Entraînement des Modèles

# Entraîner tous les modèles
python scripts/train.py --data data/raw/votre_dataset.csv

# Ou utiliser Make
make train

Faire des Prédictions

# Ligne de commande
python scripts/predict.py --input data/raw/test_data.csv --model random_forest

# Ou utiliser Make
make predict FILE=test_data.csv

API Python

Exemple d'Utilisation

from src.models import RandomForestIDS
from src.preprocessing import DataPreprocessor

# Charger et préparer les données
preprocessor = DataPreprocessor()
df = preprocessor.load_data('data.csv')
df_features = preprocessor.extract_features(df)
X_train, X_test, y_train, y_test = preprocessor.prepare_data(df_features)

# Entraîner le modèle
model = RandomForestIDS()
model.train(X_train, y_train)

# Faire des prédictions
predictions = model.predict(X_test)

Lancer le Serveur

python app.py

Docker

docker-compose logs -f

docker-compose down

Format du Dataset

Votre fichier CSV doit contenir des features de trafic réseau. Exemple:

duration,protocol_type,service,src_bytes,dst_bytes,count,label
0,tcp,http,1000,2000,5,normal
1,udp,ftp,500,1500,3,dos
...

Colonnes Communes

  • duration - Durée de la connexion
  • protocol_type - Type de protocole (tcp, udp, icmp)
  • service - Service réseau (http, ftp, ssh...)
  • src_bytes - Octets envoyés
  • dst_bytes - Octets reçus
  • count - Nombre de connexions
  • label - Étiquette (normal, dos, probe...)
